# Desktop UI 设计系统

本文件把桌面端视觉概念固化为可实现的设计约束。概念图位于
[`docs/design-assets/desktop-dashboard-concept-v1.png`](design-assets/desktop-dashboard-concept-v1.png)。

## 已知概念偏差

图片生成器在概念图中产生了两处端口文字错误。代码、测试和运行时数据必须以本表为准：

| 项目 | 正确值 | 处理 |
|---|---:|---|
| 生产入口 / Windows 系统代理 | `127.0.0.1:6666` | 本阶段只读展示，严禁接管 |
| 开发入口 | `127.0.0.1:36666` | 仅由开发 Mihomo 使用 |
| Local client A 上游 | `127.0.0.1:16666` | 当前可检测的本地出口 |
| Local client B 未来上游 | `127.0.0.1:26666` | 尚未迁移，不进行探测或控制 |

## 视觉基线

| Token | 值 | 用途 |
|---|---|---|
| `--canvas` | `#f7f9fc` | 主画布 |
| `--surface` | `#ffffff` | 表格、图表和事件区 |
| `--sidebar` | `#071426` | 左侧导航 |
| `--sidebar-active` | `#133d78` | 当前导航项 |
| `--text` | `#111827` | 主文本 |
| `--muted` | `#667085` | 次要说明 |
| `--border` | `#dce3ec` | 1px 分隔线 |
| `--accent` | `#1769e0` | 选择态、当前路径、主操作 |
| `--healthy` | `#1f9d55` | 正常 |
| `--warning` | `#e68a00` | 待迁移、退化 |
| `--danger` | `#e5484d` | 超时、不可用 |

- 字体：`Inter`, `Segoe UI`, `Microsoft YaHei UI`, sans-serif。
- 主标题：24px/32px，700；工具栏与表格：13px/20px；辅助说明：12px/18px。
- 圆角只用于控制和边界面板，默认 10px；不使用玻璃、渐变或大面积阴影。
- 布局以开放式横向轨道、数据表和时间线为主，不改造成卡片网格。

## 组件清单

| 区域 | 组件 | 状态/交互 |
|---|---|---|
| App Shell | `Sidebar`, `TopBar`, `BottomStatusBar` | 总览/历史/设置导航；当前只实现总览内容 |
| 安全边界 | `ProtectedPortBanner` | 始终显示未接管 6666；没有启用按钮 |
| 策略 | `ModeSelector` | 优先级、最低延迟、手动；未实现的策略只更新本地 UI 状态 |
| 路由 | `RouteRail` | 显示开发请求经 36666 到 16666 的链路 |
| 出口 | `OutletTable` | 行选择；状态、端口、延迟和可用率 |
| 历史 | `LatencyChart`, `EventTimeline` | 24 小时样本；最近状态变化 |
| 操作 | `CoreControls` | 只允许启动/停止由本应用创建的 36666 Mihomo 进程 |

## 首屏允许文案

`VPN Hub`、`总览`、`历史`、`设置`、`网络总览`、`最近更新`、`开发模式 · 未接管 6666`、
`测试入口 127.0.0.1:36666`、`生产入口 127.0.0.1:6666（保持不变）`、`路由模式`、
`优先级`、`最低延迟`、`手动`、`当前路由`、`出口状态`、`延迟趋势`、`最近事件`、
`Guardian`、`Mihomo`、`系统代理`。

## 容器与响应式

- 设计基准为 `1440 × 900`；最小桌面窗口 `1120 × 720`。
- 低于 1180px 时缩窄侧栏并隐藏品牌副文字；表格允许横向滚动，不能挤压列标题。
- 本阶段不设计手机布局：这是 Windows 桌面工具，但浏览器 QA 仍验证窄视口不会溢出或遮挡主操作。

## 动效

- 刷新时仅旋转刷新图标并降低内容透明度，时长 240ms。
- 图表首次加载使用 320ms 描边显现；遵守 `prefers-reduced-motion`。
- 状态变化只使用颜色与简短过渡，不闪烁。
