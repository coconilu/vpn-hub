# 已确认决策

状态：Accepted

日期：2026-07-18（最近更新：2026-07-22）

以下决策已经 Review，除非后续兼容性验证给出相反证据，否则实现按此执行。

| 编号 | 决策 | 理由 |
|---|---|---|
| D-001 | VPN Hub 永久独占 `127.0.0.1:6666` | 保持现有浏览器、IDE、终端等应用配置不变 |
| D-002 | 两个外部客户端迁移到 `16666`、`26666` | 消除端口冲突，并把它们降级为黑盒本地出口 |
| D-003 | 默认优先级为订阅 → 客户端 A → 客户端 B | 与当前人工使用习惯一致，行为可预测 |
| D-004 | 第一版采用系统代理，不启用统一 TUN | 先降低兼容性和权限复杂度 |
| D-005 | 所有出口失效时 Fail Closed | 防止用户误以为流量仍经过代理却已静默直连 |
| D-006 | 恢复需连续成功 3 次，且切换后冷却 5 分钟 | 避免抖动导致频繁来回切换 |
| D-007 | 第三方客户端按黑盒 SOCKS5 出口接入 | 当前没有证据证明其 REST API 或私有 IPC 可稳定管理 |
| D-008 | 第一版不逆向、不做 UI 自动化 | 降低版本脆弱性、合规风险和维护成本 |
| D-009 | 历史数据只记录出口质量和状态 | 不采集域名、URL、目标 IP 或用户流量内容 |
| D-010 | 先通过 Phase 0 再进入正式开发 | 端口可修改和并行运行是方案成立的硬门槛 |
| D-011 | 运行中设置重载只允许停止可同时证明 PID、Supervisor authority 与 Controller ownership 的自管核心；配置在首次 Guardian 权威回读前保持可回滚 | 防止误停第三方进程，并确保提交后启动失败不会把无效配置提升为最后有效配置 |
| D-012 | 主界面只暴露当前可完成的核心任务；手动外部 UDP 验证退出总览，不可用 TUN 退出设置页，活动出口安全选择只在删除或停用动作中出现 | 降低普通用户理解成本，同时保留 UDP `unknown/tcp_only → REJECT`、替代出口和 Fail Closed 的底层安全边界 |

## 变更规则

影响 D-001、D-005、D-009、D-011 的修改必须新建 ADR，说明动机、替代方案、安全影响和迁移方式；不能在普通重构中隐式改变。
