# 开发安全边界

## 当前硬约束

开发期间 `127.0.0.1:6666` 仍由用户当前使用的客户端占用。任何开发命令、测试、应用启动或异常清理都不得：

- 绑定、关闭、重启或修改 `6666` 的监听进程；
- 修改 Windows 系统代理；
- 开启或关闭任何第三方客户端的 TUN；
- 把系统流量切换到未验证的开发核心；
- 终止当前占用 `6666` 的第三方核心进程。

## 端口分配

| 用途 | 生产目标 | 当前开发值 |
|---|---:|---:|
| VPN Hub 对外 Mixed Port | `6666` | `36666` |
| 本地客户端 A | `16666` | `16666` |
| 本地客户端 B | `26666` | 暂不接入；当前仍占用 `6666` |
| VPN Hub 控制 API | `19090` | `39090` |

生产端口是不可变产品约束；开发值只是隔离测试配置，不得写回生产默认值。

## 测试规则

1. 所有真实出口测试必须在命令中显式指定代理地址。
2. 开发核心只监听 `127.0.0.1`，默认不设置系统代理。
3. 自动化测试使用随机临时端口，不能假定 `6666` 可用。
4. 运行前检查端口所有者，运行后只清理由当前测试进程创建的监听。
5. 订阅 URL、账号、Token、节点地址和运行数据库不得提交 Git。

## 凭据处理

兼容性检查发现，至少一个第三方客户端会把登录凭据和认证 Token 保存在本地偏好文件中。因此：

- 禁止扫描结果原样写入日志、Issue 或 CI artifact；
- 诊断工具必须采用字段白名单输出，而不是输出整个配置对象；
- VPN Hub 正式实现使用 Windows Credential Manager 或 DPAPI；
- 健康历史数据库不得包含订阅 URL、认证头或第三方客户端配置原文。
