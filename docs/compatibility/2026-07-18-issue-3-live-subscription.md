# Issue #3 受控现场验收：真实订阅与双出口

范围：VPN Hub v0.2 开发核心，入口 `127.0.0.1:36666`

结论：优先级模式的订阅 A 链路与手动超实惠链路已完成真实外网请求验收；最低延迟模式和故障注入仍未验收。

本文只记录脱敏结果，不包含订阅 URL/token、provider 节点名、Controller secret 或 provider 原文。

## 优先级模式

现场 UI 显示当前优先出口为订阅 A。健康检测中，订阅 A 延迟约 `53–54 ms`、在线率 `100%`；超实惠延迟约 `838–906 ms`、在线率约 `60%`。历史记录中已有真实切换事件。

下表是同一现场的端到端请求结果。`36666` 表示通过 VPN Hub，`16666` 是超实惠端口对照；时间为单次观测值，不代表长期性能基准。

| 目标 | 经 `36666` | 耗时 | 经 `16666` 对照 | 耗时 |
|---|---:|---:|---:|---:|
| Google | 204 | 0.295 s | 204 | 1.939 s |
| Gstatic | 204 | 0.228 s | 204 | 1.576 s |
| GitHub | 200 | 0.444 s | 200 | 2.310 s |
| Baidu | 200 | 0.386 s | 200 | 0.073 s |

结论：在优先级模式下，`36666` 已通过真实外网请求证明可经订阅 A 出口工作。上述对照只证明当次请求差异，不用于推断所有目标或所有时段的速度。

## 手动模式

用户通过 VPN Hub UI 切换为默认手动出口超实惠后，现场获得以下一致证据：

| 证据 | 现场结果 |
|---|---|
| UI | 显示 `manual · 超实惠` |
| 切换记录 | 存在原因 `manual_selection` 的 `route_switch` 事件 |
| Controller | `VPN-HUB-MASTER` 确认为 `VPN-HUB-CHAOSHIHUI` |

| 目标 | 经 `36666` | 耗时 |
|---|---:|---:|
| Google | 204 | 1.979 s |
| Gstatic | 204 | 0.986 s |
| Baidu | 200 | 0.059 s |

结论：UI 模式、持久化切换事件、Controller 实际选择和端到端外网请求四类证据一致，手动超实惠链路已确认。
